# EventHub AI 智能助手

## Goal

为已登录的用户端用户提供全站可用的 AI 智能助手，支持流式多轮对话，并基于 EventHub 的真实、受控数据完成活动推荐、场次与选座引导、本人已支付订单查询和本人票券状态查询。

AI 助手只提供只读查询与导航能力。DeepSeek 不直接访问数据库、不调用任意站内接口，也不能执行创建订单、支付、取消订单或核销票券等写操作。

本功能交付后，项目稳定版本升级至 `0.6.0`。

## Scope

- 用户端访客可在全站右下角看到并打开 AI 助手，登录后才可发起对话和访问本人业务数据。
- 支持流式回答、停止生成、重新发送和多轮上下文。
- 支持会话的新建、分页列表、切换、重命名和删除。
- 会话及完整生成成功的助手回复永久保存。
- AI 可通过后端固定工具查询：
  - 已发布活动的搜索和推荐。
  - 活动未来可售场次及选座入口。
  - 当前用户本人已支付订单。
  - 当前用户本人订单下的票券。
  - 当前用户本人票券列表和状态。
- 活动、场次、订单和票券以独立资源卡片展示，并使用后端生成的站内相对链接。
- DeepSeek Key 仅通过后端运行环境变量注入。
- 更新 OpenAPI、环境变量示例、README、架构约束和版本文档。

## Non-goals

- 匿名用户仅可查看助手介绍和登录引导，不开放对话、历史或业务查询；登录页、注册页和管理端不显示助手。
- 不提供无限制的通用聊天，回答范围限定为活动生活咨询及 EventHub 站内业务。
- 不允许 AI 创建订单、支付、取消订单、退款、修改活动、核销票券或执行其他写操作。
- 不允许模型直接访问数据库、内部网络、任意 URL 或动态注册工具。
- 不向模型发送手机号、密码、Access Token、Refresh Token、票号、二维码凭证或其他认证秘密。
- 不信任或直接渲染模型生成的 URL、HTML 或 Markdown。
- 不在前端、数据库、日志、Git 历史或设计文档中保存 DeepSeek Key。
- 首版不增加分钟、每日或账户级调用次数配额。

## Impact

### Database

新增 Flyway V9，创建以下表：

#### `eh_ai_conversation`

| 字段 | 说明 |
| --- | --- |
| `id` | 会话主键 |
| `user_id` | 所属用户，外键关联 `eh_user` |
| `title` | 会话标题，最长 100 字 |
| `created_at` | 创建时间 |
| `updated_at` | 更新时间 |
| `last_message_at` | 最后消息时间 |

建立 `(user_id, last_message_at, id)` 索引。所有读取、更新和删除操作必须校验当前用户归属。

#### `eh_ai_message`

| 字段 | 说明 |
| --- | --- |
| `id` | 消息主键 |
| `conversation_id` | 所属会话，删除会话时级联硬删除 |
| `role` | `USER` 或 `ASSISTANT` |
| `content` | 纯文本消息正文 |
| `resources_json` | 后端生成的资源卡片 JSON |
| `model` | 完成回复所使用的模型 |
| `prompt_tokens` | 输入 Token 用量 |
| `completion_tokens` | 输出 Token 用量 |
| `created_at` | 创建时间 |

建立 `(conversation_id, id)` 索引。不保存模型思考过程、工具调用参数、工具原始结果或未完整生成的助手回复。

### Public APIs

新增认证用户接口：

| Method | Path | Purpose |
| --- | --- | --- |
| `GET` | `/api/assistant/conversations` | 分页查询本人会话 |
| `POST` | `/api/assistant/conversations` | 新建会话 |
| `PUT` | `/api/assistant/conversations/{conversationId}` | 重命名本人会话 |
| `DELETE` | `/api/assistant/conversations/{conversationId}` | 删除本人会话及消息 |
| `GET` | `/api/assistant/conversations/{conversationId}/messages` | 分页查询本人会话消息 |
| `POST` | `/api/assistant/conversations/{conversationId}/messages/stream` | 发送消息并流式接收回复 |

普通 CRUD 进入 OpenAPI。流式接口返回 `text/event-stream`，事件固定为：

| Event | Payload |
| --- | --- |
| `ack` | `conversationId`、已持久化的 `userMessageId` |
| `delta` | 增量纯文本 `content` |
| `resources` | 后端校验并生成的资源卡片数组 |
| `done` | `assistantMessageId`、模型及 Token 用量 |
| `error` | 错误码、用户消息和 `retryable` |

资源卡片固定包含 `type`、`id`、`title`、`subtitle`、`href`。链接仅由后端按资源类型生成：

- 活动：`/activities/{id}`
- 场次：`/sessions/{id}/tickets`
- 订单：`/orders/{id}`
- 票券：`/tickets/{id}`

### Security And Privacy

- 所有助手接口要求 `USER` 角色。
- 会话、订单和票券查询全部使用当前 `AuthenticatedUser.id` 约束归属。
- 订单号输入仅用于后端本人数据匹配，发送给模型时使用内部 ID 或脱敏显示值。
- 发送给模型的票券信息只包含活动、场次、场馆、座位、支付状态、票券状态和核销时间。
- 工具结果一律视为不可信数据，系统提示词明确禁止执行其中的指令。
- 模型输出的业务事实只有在工具结果中存在时才可作为实时站内信息回答。
- 资源链接由后端建立，模型正文中的 URL 不作为可点击链接。
- 聊天正文按纯文本展示，不使用 `dangerouslySetInnerHTML`。

### Runtime Configuration

后端新增以下环境变量：

```text
DEEPSEEK_API_KEY=
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-v4-flash
```

未配置 Key 时返回 `AI_NOT_CONFIGURED`，不得打印 Key、请求授权头或完整敏感业务载荷。

DeepSeek 使用 OpenAI 兼容的 `/chat/completions` 接口，启用流式输出和 Tool Calls，关闭思考模式。应用负责发送多轮历史，不依赖供应商保存会话状态。

### Error Codes

| Error code | HTTP status | Meaning |
| --- | --- | --- |
| `AI_NOT_CONFIGURED` | `503` | 后端未配置 AI 服务 |
| `AI_CONVERSATION_NOT_FOUND` | `404` | 会话不存在 |
| `AI_CONVERSATION_ACCESS_DENIED` | `403` | 会话不属于当前用户 |
| `AI_STREAM_BUSY` | `409` | 当前用户已有生成任务 |
| `AI_UPSTREAM_UNAVAILABLE` | `502` | DeepSeek 不可用、超时或协议异常 |
| `AI_TOOL_LIMIT_EXCEEDED` | `502` | 模型超过允许的工具调用轮次 |

单条输入超过 2000 字等参数错误继续使用现有校验错误机制。

## Implementation Plan

### 1. Database And Conversation Management

- 新增 Flyway V9。
- 新增会话和消息实体、Mapper、DTO 及服务。
- 会话列表按 `last_message_at` 倒序分页。
- 前端首次发送前创建会话，默认标题从首条问题截取；用户可重命名。
- 删除会话时由数据库外键级联硬删除消息。
- 历史上下文仅向模型发送最近 20 轮，数据库历史不自动过期。

### 2. Assistant Backend Module

新增独立 `com.eventhub.assistant` 模块，按职责拆分：

- Controller：认证、参数校验和 SSE 协议。
- Conversation Service：会话归属、历史和消息持久化。
- Streaming Coordinator：模型调用、工具循环、取消和事件编排。
- DeepSeek Client：请求构建、流式增量及 Tool Calls 解析。
- Tool Registry/Executor：固定只读工具定义和执行。
- Resource Factory：校验资源并生成站内相对链接。
- Prompt：系统规则和数据边界，存放在后端资源 `prompts/` 目录。

生成流程：

1. 校验会话归属、输入长度和用户并发状态。
2. 持久化用户消息并发送 `ack`。
3. 读取最近 20 轮对话，调用 DeepSeek。
4. 聚合流式文本或 Tool Calls 增量。
5. 最多执行 3 轮固定工具，工具结果作为不可信上下文回传模型。
6. 发送 `delta` 和经后端生成的 `resources`。
7. 仅在完整完成后保存助手消息并发送 `done`。
8. 上游异常、超时或客户端断开时取消后续工作，不保存不完整助手消息。

技术限制：

- 单条输入最多 2000 字。
- 最多发送最近 20 轮。
- 工具调用最多连续 3 轮。
- 单次请求总超时 60 秒。
- 使用 Redis 锁限制每个用户同时只能有一个流，锁超时约 75 秒。
- 不设置分钟或每日调用次数配额。

### 3. Fixed Read-only Tools

首版只注册以下工具：

1. `search_activities`
   - 参数支持关键词、城市、分类、日期和最高预算。
   - 只返回已发布且存在未来场次的活动。
   - 最多返回 5 个真实活动及后端资源卡片。
2. `get_activity_sessions`
   - 查询指定活动的未来正常场次、有效票种、余量和选座入口。
3. `list_my_paid_orders`
   - 仅查询当前用户本人 `PAID` 订单，最多返回 10 条。
4. `get_my_order_tickets`
   - 按本人订单 ID 或订单号查询票券，先验证订单归属。
5. `list_my_tickets`
   - 按状态查询当前用户本人票券，最多返回 20 条。

工具不接受用户 ID，不支持任意 SQL、任意接口、动态 URL 或任何写操作。

### 4. DeepSeek Integration

- 使用后端 HTTP 客户端连接 DeepSeek，不将供应商 SDK 暴露到业务模块。
- 解析流式 `content` 和分片 Tool Calls，并对上游非成功状态、无效 JSON、提前断流和超时统一映射错误。
- 使用本地模拟 HTTP 服务覆盖多轮 Tool Calls 和流式协议测试，CI 不依赖真实 Key。
- 使用有界执行器承载流式任务，客户端断开时取消上游请求并释放 Redis 锁。
- 系统提示词禁止泄露系统规则、执行工具数据中的指令、伪造实时库存或返回未验证业务链接。

### 5. Frontend Assistant

- 在 `AppProviders` 的认证上下文内挂载全局助手。
- 仅当前登录主体为 `USER_WEB` 时显示；登录、注册和管理端不显示。
- 桌面端使用右下固定品牌按钮和约 420px 宽对话面板。
- 移动端使用全屏抽屉，保留关闭、历史切换和停止生成操作。
- 支持新对话、历史分页、切换、重命名、删除、停止生成、自动滚动和重新发送。
- 提供快捷问题：
  - 推荐周末活动
  - 查看我的待使用票券
  - 查询最近已支付订单
- 消息正文按安全纯文本显示；资源卡片通过 React Router 跳转。
- 普通会话 CRUD 使用重新生成的 OpenAPI 客户端。
- 流式消息使用 `fetch`、Bearer Token、`ReadableStream` 和 `AbortController`。
- 收到 `401` 时调用现有 Token 刷新机制并最多重试一次。
- 退出登录时终止流式请求并清空内存中的助手状态。

## Risks

- 未设置调用配额会带来供应商成本不可控风险；首版通过并发、长度、轮次和超时限制控制单次消耗，但不能控制账户总量。
- 外部模型可能产生幻觉；实时站内信息必须来自工具，界面需将可验证资源与自然语言分离。
- 活动余量和票券状态可能在查询后变化；回答应明确以详情页当前状态为准。
- 工具结果和历史消息可能包含提示注入文本；必须按不可信数据处理并限制工具能力。
- SSE 在代理、浏览器切页和弱网环境下可能提前断开；服务端必须及时取消上游请求和释放并发锁。
- 会话永久保存会持续增长数据库容量，后续需要依据实际使用量补充归档或用户导出策略。
- DeepSeek 模型名、协议或能力可能调整；模型和 Base URL 必须可配置，协议解析需要隔离测试。
- 将业务数据发送给第三方模型存在数据处理风险；首版通过严格脱敏和最小字段集降低暴露面。

## Rollback Plan

- 前端移除或关闭全局助手挂载，不影响现有活动、订单和票券页面。
- 后端禁用助手接口或移除 `DEEPSEEK_API_KEY`，核心业务继续工作。
- 回滚助手模块和 OpenAPI 变更时保留 V9 数据表，避免生产数据库执行破坏性降级。
- 如后续确认不再使用，在备份会话数据后单独安排数据库清理迁移。
- 助手工具仅调用现有只读能力，因此回滚不需要补偿订单、支付或票券数据。
