# EventHub 已发布活动内容编辑

## Goal

允许商家在活动审核发布后继续维护不影响交易和履约的展示内容，包括活动封面、一句话介绍和活动详情，并确保公开活动缓存及时失效、并发修改不会覆盖他人更新。

## Scope

- 复用现有 `PUT /api/merchant/activities/{activityId}` 接口。
- `DRAFT`、`REJECTED` 活动继续允许修改全部活动基本信息。
- `PUBLISHED` 活动只允许修改：
  - `coverUrl`
  - `summary`
  - `description`
- 已发布活动修改后保持 `PUBLISHED`，无需重新提交审核。
- 已发布活动的标题、分类和城市必须保持不变。
- 已发布活动的场次、场馆、售卖时间、票档、价格、库存和座位配置继续禁止修改或删除。
- 管理端活动列表增加“编辑信息”入口。
- 编辑窗口加载活动详情并回填现有内容，支持重新上传或移除封面。
- 更新成功后刷新商家活动列表并使公开活动详情缓存失效。
- 保留现有乐观锁版本校验。

## Non-goals

- 不允许已发布活动修改标题、分类或城市。
- 不允许已发布活动新增、修改或删除场次和票档。
- 不增加已发布内容二次审核流程。
- 不保存活动内容修改历史。
- 不增加管理员审批、撤销或版本回滚功能。
- 不修改数据库结构。
- 不自动删除被替换或移除的历史封面文件。

## Impact

### 后端行为

现有更新接口的请求和响应结构保持不变：

```text
PUT /api/merchant/activities/{activityId}
```

- 对 `DRAFT`、`REJECTED` 保持现有完整更新行为。
- 对 `PUBLISHED` 使用独立的展示内容更新语句，只写入 `summary`、`description`、`cover_url` 和递增后的 `version`。
- 后端不能依赖前端禁用字段保证安全；收到已发布活动更新请求时，必须校验请求中的标题、分类和城市与数据库当前值一致。
- 更新条件必须包含活动编号、商家编号、`PUBLISHED` 状态和当前版本。
- 更新成功后删除 `eventhub:activity:detail:{activityId}` 缓存，使公开页面下一次查询读取最新内容。

### 管理端

- 将现有创建活动表单整理为可复用的活动信息表单或新增独立编辑窗口。
- 商家活动列表中的 `DRAFT`、`REJECTED`、`PUBLISHED` 活动均显示“编辑信息”。
- 编辑 `PUBLISHED` 活动时，标题、分类和城市只读；封面、简介和详情可编辑。
- 编辑草稿或已驳回活动时，保持全部基本信息可编辑。
- 保存时提交详情接口返回的最新 `version`，版本冲突沿用现有统一错误提示。

### 稳定约束

活动领域约束调整为：

- 商家只能完整编辑 `DRAFT` 或 `REJECTED` 活动。
- 商家可以编辑 `PUBLISHED` 活动的封面、简介和详情。
- 已发布活动中影响交易、检索归类和履约的数据仍不可直接修改。

本变更不调整数据库结构，不改变现有接口路径和数据结构，但会改变已发布活动更新接口的允许行为，因此需要同步更新长期架构约束和版本记录。

## Implementation plan

1. 扩展活动状态规则，区分完整编辑和已发布展示内容编辑。
2. 在活动命令 Mapper 增加已发布展示内容的条件更新。
3. 调整商家活动更新服务，根据当前状态选择完整更新或展示内容更新。
4. 为已发布活动不可变字段校验、允许字段更新、越权和版本冲突补充测试。
5. 在管理端 API 封装中接入现有活动更新接口。
6. 增加活动信息编辑窗口，回填详情并按状态限制字段。
7. 在商家活动列表增加编辑入口并刷新查询缓存。
8. 重新生成或核对 OpenAPI 客户端，确保生成目录不手工修改。
9. 更新架构约束、版本记录和 README 功能说明。
10. 运行后端测试、两个前端检查、`git diff --check` 并检查最终差异。

## Risks

- 若后端只依赖前端禁用字段，调用者可能绕过页面修改已发布活动关键字段，因此必须在服务端校验和使用受限 SQL。
- 已发布内容更新后若缓存未失效，用户端可能继续展示旧封面或描述。
- 两个商家员工同时编辑可能发生覆盖，必须继续使用 `version` 乐观锁。
- 替换封面后旧文件仍会保留，可能形成孤立文件；本次不引入文件引用清理任务。
- 已发布内容不经过二次审核，商家可能发布不合规描述；当前按用户要求采用直接更新，后续可扩展内容审核和审计。

## Rollback plan

- 恢复状态机只允许 `DRAFT`、`REJECTED` 完整编辑。
- 移除已发布展示内容更新 Mapper 和服务分支。
- 移除管理端已发布活动“编辑信息”入口。
- 已经修改的封面和描述保留在数据库中，不执行数据回滚。
- 无数据库迁移，应用代码回滚即可恢复原行为。
